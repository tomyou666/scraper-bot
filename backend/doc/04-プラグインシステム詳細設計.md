## 概要

- Plugin interface 群、種類別 interface（PreProcessor / Parser / Transformer / Filter / LinkExtractor）、Registry、`init()` 登録方式、`Host` interface、ライフサイクル管理を確定する。
- 採用方式は **コンパイル時組み込み + `init()` 登録**。`cmd/scraperbot/main.go` の副作用 import で機能セットを決める。
- 本章で確定する interface 群は、`internal/domain/plugin/` に置き、`core` / `usecase` / `plugins/*` 全てがこの抽象だけに依存する。

## 詳細

### 共通 Plugin / Host interface

すべてのプラグインに共通する最小契約。種類別 interface はこれを埋め込む。

```go
// internal/domain/plugin/plugin.go
package plugin

import (
    "context"
    "net/url"
)

type Kind string

const (
    KindPreProcessor  Kind = "preprocessor"   // P2
    KindParser        Kind = "parser"         // P5
    KindTransformer   Kind = "transformer"    // P6
    KindFilter        Kind = "filter"         // P7
    KindLinkExtractor Kind = "link_extractor" // P8
    KindFetcher       Kind = "fetcher"        // URL 取得
)

type Metadata struct {
    Name        string
    Version     string
    Kind        Kind
    Description string
}

type Plugin interface {
    Metadata() Metadata
    Init(ctx context.Context, host Host) error
    Close(ctx context.Context) error
}

type Host interface {
    Config(key string) (string, bool)
    RequestConfig() model.RequestConfig
    FetcherConfig() model.FetcherConfig
}


```

- `Host` を介してプラグインへ設定リソースを渡す。グローバル変数は原則使わない（**例外**: ログは `internal/logger` で `slog.SetDefault` により Go 標準ロガーとしてグローバル初期化し、`slog.Info` / `slog.Warn` 等を直接呼ぶ）。
- ログ実装は `internal/logger` に集約し、`log/slog` の `TextHandler` を既定とする。
- プラグイン向けの HTTP モデルは `plugin.HTTPRequest` / `plugin.HTTPResponse` を使う。スクレイピング本体の `model.Request` / `model.Response` とは分離し、責務を混在させない。

### 共通リクエスト / レスポンスモデル

P2 が触れるリクエスト、P5 が受け取るレスポンス、後段が扱うコンテンツの3層に分けて定義する。

```go
// internal/domain/model/request.go
package model

import (
    "net/url"
    "time"
)

type Request struct {
    URL     *url.URL
    Method  string
    Headers map[string]string
    Depth   int
    Meta    map[string]any
}

// internal/domain/model/response.go
type Response struct {
    URL         *url.URL
    StatusCode  int
    Headers     map[string]string
    ContentType string
    Body        []byte
    FetchedAt   time.Time
}

// internal/domain/model/content.go
type Content struct {
    URL         *url.URL
    Format      string // "html", "pdf", "text", "json"
    Text        string // 構造化テキスト（HTMLマークダウン化前の中間表現や PDF テキストなど）
    DOM         any    // HTMLパーサーが選択的に格納する内部表現（例: *goquery.Document）
    Metadata    map[string]string
    Attachments []Attachment // 例: PDFのページごとの画像など
}

type Attachment struct {
    URL  *url.URL
    Kind string // "pdf", "image", ...
    Data []byte
}

// internal/domain/model/result.go
type Result struct {
    URL      *url.URL
    Markdown string
    HTML     string
    RawHTML  string
    JSON     map[string]any
    Links    []*url.URL
    Metadata map[string]string
}
```

### 種類別 Plugin interface

採用したプラグイン実行ポイント P2 / P5 / P6 / P7 / P8 に対応する。

```go
// internal/domain/plugin/preprocessor.go
package plugin

import (
    "context"
    "scraperbot/internal/domain/model"
)

// P2: リクエスト前処理
type PreProcessor interface {
    Plugin
    PreProcess(ctx context.Context, req *model.Request) error
}
```

```go
// internal/domain/plugin/parser.go
package plugin

import (
    "context"
    "scraperbot/internal/domain/model"
)

// P5: コンテンツ解析
type Parser interface {
    Plugin
    // CanParse はレスポンスを処理できるか判定。Content-Type やURL拡張子で判定。
    CanParse(res *model.Response) bool
    Parse(ctx context.Context, res *model.Response) (*model.Content, error)
}
```

```go
// internal/domain/plugin/transformer.go
package plugin

import (
    "context"
    "scraperbot/internal/domain/model"
)

// P6: コンテンツ変換
type Transformer interface {
    Plugin
    Transform(ctx context.Context, c *model.Content) (*model.Result, error)
}
```

```go
// internal/domain/plugin/filter.go
package plugin

import (
    "context"
    "scraperbot/internal/domain/model"
)

// P7: コンテンツフィルタリング
type Filter interface {
    Plugin
    Filter(ctx context.Context, c *model.Content) (*model.Content, error)
}
```

```go
// internal/domain/plugin/linkextractor.go
package plugin

import (
    "context"
    "net/url"
    "scraperbot/internal/domain/model"
)

// P8: リンク抽出・次ページ決定
type LinkExtractor interface {
    Plugin
    Extract(ctx context.Context, c *model.Content, baseURL *url.URL) ([]*url.URL, error)
}
```

### P3 Fetcher プラグイン（URL 取得）

- パイプライン上は **P3**（P2 PreProcessor の次、P5 Parser の前）。
- 抽象は `plugin.Fetcher`（`Plugin` + `Get(ctx, *url.URL, headers)`）。
- 登録 API: `core.RegisterFetcher(name, func() plugin.Fetcher)`
- 生成・初期化: `Kernel.Init` が `NewFetcher(plugins.fetcher)` → `Init(ctx, host)` を P2 の直後に実行。
- ビルトイン:
    - `plugins/fetcher-http` → 名前 `http`（net/http 実装をプラグイン内に保持）
    - `plugins/fetcher-chromium` → 名前 `chromium`（chromedp 実装をプラグイン内に保持）
- `Host.HTTP()` は廃止。補助 HTTP が必要な場合は P3 Fetcher の `Get` を使うか、将来専用プラグインを追加する。
- robots.txt 取得は `robots.NewCache(kernel.Fetcher())` でページと同一 Fetcher を使用。

```go
// plugins/fetcher-http/plugin.go（例）
func init() {
    core.RegisterFetcher("http", func() plugin.Fetcher { return &client{} })
}
```

### Registry 設計

- レジストリは種類別に分かれた map を持ち、`init()` から `RegisterXxx` 関数で登録される。
- レジストリは具体プラグインを `import` しない。プラグイン側が逆向きに依存する。

```go
// internal/core/registry.go
package core

import (
    "fmt"
    "sync"

    "scraperbot/internal/domain/plugin"
)

type Registry struct {
    mu sync.RWMutex
    preprocessors  map[string]func() plugin.PreProcessor
    parsers        map[string]func() plugin.Parser
    transformers   map[string]func() plugin.Transformer
    filters        map[string]func() plugin.Filter
    linkextractors map[string]func() plugin.LinkExtractor
    fetchers       map[string]func() plugin.Fetcher
}

var defaultRegistry = newRegistry()

func newRegistry() *Registry {
    return &Registry{
        preprocessors:  map[string]func() plugin.PreProcessor{},
        parsers:        map[string]func() plugin.Parser{},
        transformers:   map[string]func() plugin.Transformer{},
        filters:        map[string]func() plugin.Filter{},
        linkextractors: map[string]func() plugin.LinkExtractor{},
    }
}

func Default() *Registry { return defaultRegistry }

func RegisterPreProcessor(name string, f func() plugin.PreProcessor) {
    defaultRegistry.mu.Lock()
    defer defaultRegistry.mu.Unlock()
    if _, dup := defaultRegistry.preprocessors[name]; dup {
        panic(fmt.Sprintf("preprocessor already registered: %s", name))
    }
    defaultRegistry.preprocessors[name] = f
}

// RegisterParser / RegisterTransformer / RegisterFilter / RegisterLinkExtractor / RegisterFetcher も同様

func (r *Registry) NewFetcher(name string) (plugin.Fetcher, error) { /* ... */ }

func (r *Registry) NewPreProcessor(name string) (plugin.PreProcessor, error) {
    r.mu.RLock()
    defer r.mu.RUnlock()
    f, ok := r.preprocessors[name]
    if !ok {
        return nil, fmt.Errorf("preprocessor not found: %s", name)
    }
    return f(), nil
}

// NewParser / NewTransformer / NewFilter / NewLinkExtractor も同様
```

- ファクトリ関数（`func() plugin.Xxx`）で登録する理由は、プラグインのインスタンスを **カーネル起動時に新規生成して `Init` するため**。シングルトンを `init()` で作ると `Host` を渡せない。
- 同名重複登録は `panic`（ビルド時の事故を起動直後に検出するため）。

### init() 登録方式

各プラグインパッケージは `init()` 内で自身を登録する。

```go
// plugins/transformer-markdown/plugin.go
package markdown

import (
    "context"

    "scraperbot/internal/core"
    "scraperbot/internal/domain/model"
    "scraperbot/internal/domain/plugin"
)

func init() {
    core.RegisterTransformer("markdown", func() plugin.Transformer {
        return &transformer{}
    })
}

type transformer struct {
    host plugin.Host
}

func (t *transformer) Metadata() plugin.Metadata {
    return plugin.Metadata{
        Name:        "markdown",
        Version:     "0.1.0",
        Kind:        plugin.KindTransformer,
        Description: "Convert HTML content to Markdown",
    }
}

func (t *transformer) Init(ctx context.Context, host plugin.Host) error {
    t.host = host
    return nil
}

func (t *transformer) Close(ctx context.Context) error { return nil }

func (t *transformer) Transform(ctx context.Context, c *model.Content) (*model.Result, error) {
    // HTML -> Markdown 変換
    return &model.Result{ /* ... */ }, nil
}
```

### 同名種類で複数選択する場合

設定 `PluginSelection` で、複数選択可（PreProcessor / Parser / Filter）と単一選択（Transformer / LinkExtractor）を区別する。

- 複数選択（P2 / P5 / P7）
  - 設定で指定された名前順にチェーンとして実行する。
  - P5（Parser）のみ例外: 設定で複数登録するが、実際の選択はレスポンスごとに `CanParse(res)` で決まる。HTML / PDF を両方登録し、Content-Type で振り分けるイメージ。
- 単一選択（P6 / P8）
  - 設定で1つだけ指定。未指定はデフォルト値（`markdown` / `default`）。

### Host 実装

`Host` の実体は `core` 側にある。プラグインからの参照は interface 越し。

```go
// internal/core/host.go
package core

import (
    "scraperbot/internal/domain/model"
    "scraperbot/internal/domain/plugin"
)

type hostImpl struct {
    cfg *model.Config
}

func (h *hostImpl) Config(key string) (string, bool) {
    // flat keyで取り出す。詳細は `internal/core/configview.go`。
    return lookupFlat(h.cfg, key)
}
```

- `Config(key)` は `request.headers.User-Agent` のような flat key で値を取れる軽量 API。プラグインに `*model.Config` を直接渡さない（ドメイン的に過剰な権限を与えないため）。

### ログ

```go
// internal/logger/logger.go
package logger

func Init(w io.Writer, level slog.Level) {
    h := NewHandler(w, level)
    slog.SetDefault(slog.New(h))
}
```

- CLI 入口（`cli.RunApp`）で `logger.Init(stderr, slog.LevelInfo)` を 1 回呼び、Go 全体の標準ロガーを差し替える。
- core / infrastructure / plugins は `slog.Info` / `slog.Warn` / `slog.Error` を直接呼ぶ。Host 経由の DI は行わない。
- Markdown 本体などアプリ出力（データ）は stdout へ直接書き込み、ログとは分離する。

### Kernel と ライフサイクル

```go
// internal/core/kernel.go
package core

import (
    "context"
    "errors"
    "fmt"

    "scraperbot/internal/domain/model"
    "scraperbot/internal/domain/plugin"
)

type Kernel struct {
    cfg            *model.Config
    host           plugin.Host
    preprocessors  []plugin.PreProcessor
    parsers        []plugin.Parser
    transformer    plugin.Transformer
    filters        []plugin.Filter
    linkExtractor  plugin.LinkExtractor
}

func NewKernel(cfg *model.Config, host plugin.Host) *Kernel {
    return &Kernel{cfg: cfg, host: host}
}

func (k *Kernel) Init(ctx context.Context) error {
    reg := Default()
    inited := make([]plugin.Plugin, 0, 8)
    rollback := func(initErr error) error {
        var errs []error
        for i := len(inited) - 1; i >= 0; i-- {
            if err := inited[i].Close(ctx); err != nil {
                errs = append(errs, err)
            }
        }
        if len(errs) == 0 {
            return initErr
        }
        return errors.Join(append([]error{initErr}, errs...)...)
    }

    for _, name := range k.cfg.Plugins.PreProcessors {
        p, err := reg.NewPreProcessor(name)
        if err != nil {
            return rollback(err)
        }
        if err := p.Init(ctx, k.host); err != nil {
            return rollback(fmt.Errorf("init preprocessor %s: %w", name, err))
        }
        k.preprocessors = append(k.preprocessors, p)
        inited = append(inited, p)
    }

    for _, name := range k.cfg.Plugins.Parsers {
        p, err := reg.NewParser(name)
        if err != nil {
            return rollback(err)
        }
        if err := p.Init(ctx, k.host); err != nil {
            return rollback(fmt.Errorf("init parser %s: %w", name, err))
        }
        k.parsers = append(k.parsers, p)
        inited = append(inited, p)
    }

    t, err := reg.NewTransformer(k.cfg.Plugins.Transformer)
    if err != nil {
        return rollback(err)
    }
    if err := t.Init(ctx, k.host); err != nil {
        return rollback(fmt.Errorf("init transformer %s: %w", k.cfg.Plugins.Transformer, err))
    }
    k.transformer = t
    inited = append(inited, t)

    for _, name := range k.cfg.Plugins.Filters {
        p, err := reg.NewFilter(name)
        if err != nil {
            return rollback(err)
        }
        if err := p.Init(ctx, k.host); err != nil {
            return rollback(fmt.Errorf("init filter %s: %w", name, err))
        }
        k.filters = append(k.filters, p)
        inited = append(inited, p)
    }

    le, err := reg.NewLinkExtractor(k.cfg.Plugins.LinkExtractor)
    if err != nil {
        return rollback(err)
    }
    if err := le.Init(ctx, k.host); err != nil {
        return rollback(fmt.Errorf("init linkextractor %s: %w", k.cfg.Plugins.LinkExtractor, err))
    }
    k.linkExtractor = le
    inited = append(inited, le)

    return nil
}

func (k *Kernel) Close(ctx context.Context) error {
    // 登録の逆順でClose
    var errs []error
    if k.linkExtractor != nil {
        errs = append(errs, k.linkExtractor.Close(ctx))
    }
    for i := len(k.filters) - 1; i >= 0; i-- {
        errs = append(errs, k.filters[i].Close(ctx))
    }
    if k.transformer != nil {
        errs = append(errs, k.transformer.Close(ctx))
    }
    for i := len(k.parsers) - 1; i >= 0; i-- {
        errs = append(errs, k.parsers[i].Close(ctx))
    }
    for i := len(k.preprocessors) - 1; i >= 0; i-- {
        errs = append(errs, k.preprocessors[i].Close(ctx))
    }
    return joinNonNil(errs)
}
```

- 起動順: PreProcessors → Parsers → Transformer → Filters → LinkExtractor。
- 停止順: その逆。
- いずれかの `Init` 失敗は致命扱いで起動中止。失敗時はそれまでに `Init` 済みのプラグインを逆順で `Close` してロールバックする。
- `Close` のエラーは集約（`errors.Join`）して返すが、起動失敗扱いにはしない（クリーンアップ時のため）。

### エラーポリシーまとめ

| フェーズ | 失敗時の挙動 |
| --- | --- |
| プラグイン `Init` | 致命。プロセス終了。 |
| プラグイン `Close` | 集約してログ。プロセスはそのまま終了。 |
| `PreProcess` | 当該ページのみ失敗扱い。次ページの処理は継続。 |
| `Parse` | パーサーが見つからない／全て失敗 → 当該ページのみ失敗扱い。 |
| `Transform` / `Filter` | 当該ページのみ失敗扱い。 |
| `LinkExtractor` | 失敗してもページの結果出力は行う。後続キューに追加されないだけ。 |

### テスト方針

- `domain/plugin/*` interface に対し、`core/registry_test.go` でダミー実装を登録 → 取得できることを単体テスト。
- 各プラグインは `Host` をモック化（`plugin.Host` のフェイク実装）して独立にテスト。
- カーネルは「全 Init 成功」「途中 Init 失敗で前段がCloseされること」「Close が登録の逆順で呼ばれること」をテーブル駆動でテスト。

### 受け入れ条件

- `internal/domain/plugin/` 以下が外部依存ゼロ（Go標準ライブラリのみ）。
- `core.Default()` のレジストリに、起動前は何も登録されていない（テスト用に分離可能）。
- `init()` 登録は同名重複で panic する。
- カーネルの `Init` 失敗時、それまでに成功した `Init` は逆順で `Close` される。
- 設定で指定した名前のプラグインがレジストリにない場合、起動前に明示的エラー。
