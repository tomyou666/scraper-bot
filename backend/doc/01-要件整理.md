## 概要

- Go言語で実装する、コンパイル時プラグイン組み込み型のスクレイピングツールの要件を定義する。
- 対象は単一ページの取得から、リンクを辿る複数ページの巡回（クロール）、およびリンク先PDFのテキスト抽出までを含む。
- 実行は当面 `main` 関数による動作確認とし、将来的に Fyne デスクトップアプリ化することを前提とする。
- 本章は後続章（アーキテクチャ、設定、プラグイン、パイプライン）に対する制約条件を確定する位置づけ。

## 詳細

### 目的とゴール

- 目的: Webページとリンク先PDFを取得し、機械処理しやすい形（Markdown/HTML/構造化JSON）に変換するツールを作る。
- ゴール:
  - 単一URLのスクレイピングが CLI（`main` 直接実行）で動作する。
  - クロール設定（深度・件数・パス制限）で複数ページを巡回できる。
  - リンクされた PDF のテキスト抽出が可能。
  - スクレイピング処理の主要ポイントを **コンパイル時プラグイン** として差し替え可能。
  - プレゼンテーション層の差し替えで、後日 Fyne GUI に載せ替え可能。

### 想定ユースケース

| # | ユースケース | 入力 | 出力 |
| --- | --- | --- | --- |
| UC-1 | 単一ページの Markdown 化 | 1つのURL | Markdown文字列とメタデータ |
| UC-2 | サイト内クロール | 起点URL + クロール設定 | 各ページの変換結果一覧 |
| UC-3 | リンク先PDF抽出 | URL（ページ内に PDF リンクを含む） | 各PDFのテキスト/Markdown |
| UC-4 | CSSセレクタ指定抽出 | URL + セレクタ | 一致要素から抽出した本文 |
| UC-5 | プラグイン差し替えビルド | ビルドタグ・副作用 import 構成 | 機能セットの異なるバイナリ |

### 機能要件

- リクエスト制御
  - 任意URLへのGETリクエスト送信
  - カスタム HTTP ヘッダ指定
  - タイムアウト指定
  - リトライ（回数・間隔）
- コンテンツ取得・整形
  - 出力フォーマット切り替え（Markdown / HTML / Raw HTML / JSON / Links / Metadata）
  - メインコンテンツのみ抽出（ヘッダー・フッター・ナビ除去）
  - HTMLタグ単位の include / exclude
  - CSSセレクタ指定での抽出
  - リンク一覧抽出
  - メタデータ（title / description / OGP 等）抽出
- PDF処理
  - リンク先 PDF を検出し、テキスト抽出する
  - 抽出モード: `fast`（埋め込みテキストのみ）/ `auto`（fast 失敗時 OCR）/ `ocr`（強制OCR）
  - 最大ページ数指定
  - 出力形式（テキスト / Markdown / 生バイナリ）
- クローリング
  - 最大深度
  - 最大ページ数
  - 含める/除外するパスパターン（正規表現）
  - 外部リンク追跡可否
  - サブドメイン追跡可否
  - リクエスト間遅延
  - 並行スクレイピング数
  - `robots.txt` 準拠
- 拡張性
  - リクエスト前処理、コンテンツ解析、変換、フィルタリング、リンク抽出の各ポイントでプラグインを差し替え可能
  - プラグインはコンパイル時に副作用 import で組み込む

### 非機能要件

- 言語: Go（モジュールモード、`go.mod` 管理）
- プラットフォーム: Windows / Linux / macOS（クロスコンパイル可能なバイナリ）
- 動作モード: CLI（当面）→ Fyne GUI（将来）
- 配布形態: 単一バイナリ。プラグインは出荷バイナリにビルド時で固定する。
- 拡張時のビルド: 利用者が `cmd/<app>/main.go` の副作用 import を編集して再ビルドできる。
- ログ: 標準ログ出力。プラグインから利用可能なロガーをコアが提供する。
- エラー方針: クロール中の個別ページ失敗は致命扱いしない（スキップしてログ）。設定不備や依存初期化失敗は致命とする。

### 技術スタック

| 区分 | 採用 | 理由 |
| --- | --- | --- |
| 言語 | Go 1.26 以上 | 最新安定機能を前提にし、長期保守での移植コストを下げる |
| HTTPクライアント | `net/http` | 標準ライブラリで十分。プラグインで差し替え可能 |
| HTMLパース | `github.com/PuerkitoBio/goquery` | CSSセレクタ指定をそのまま実装でき、利用例が豊富 |
| HTML→Markdown 変換 | `github.com/JohannesKaufmann/html-to-markdown`（候補） | Markdown 出力の品質と保守性 |
| PDFテキスト抽出（fast） | `github.com/ledongthuc/pdf` または `github.com/dslipak/pdf`（候補） | 埋め込みテキストの抽出が可能 |
| OCR（PDF auto/ocr モード） | Tesseract 連携（`gosseract` 等）（候補） | OCR は外部エンジンに委譲 |
| robots.txt | `github.com/temoto/robotstxt`（候補） | 標準的な robots.txt パーサー |
| 設定ファイル | YAML（`gopkg.in/yaml.v3`）（候補） | 人間可読・コメント可・階層構造に適合 |
| GUI（将来） | Fyne | クロスプラットフォーム Go ネイティブ GUI |

- 上記のうち「候補」と付したものは、詳細設計章で確定する。要件としては **抽象 interface でラップしてプラグイン差し替え可能にする** ことを必須とする。

### 前提

- ターゲットサイトは公開Web（要認証サイトは対象外、ただしカスタムヘッダで Cookie 等を持ち込むことは可能）。
- 大規模分散クロールは目的外。単一プロセス内での並行処理に限定する。
- JavaScript レンダリングが必要な動的ページは `plugins.fetcher: chromium` で対応可能（chromedp によるヘッドレス Chromium 取得）。既定は静的 HTML 向けの `http` 取得。
- Windows 環境でも改行コード `LF` で実装ファイルを管理する。

### 制約

- アーキテクチャ制約
  - システム構成は **マイクロカーネルアーキテクチャ**（コンパイル時プラグイン組み込み）。
  - フォルダ構造は **レイヤードアーキテクチャ**。
  - プレゼンテーション層は当面 CLI(`main`)、将来 Fyne に置き換え可能とする抽象境界を持つ。
- プラグイン点制約
  - プラグインを差し込めるのは以下の5点に限定する（後続章で詳細化）:
    - P2: リクエスト前処理
    - P5: コンテンツ解析（パーサー）
    - P6: コンテンツ変換
    - P7: コンテンツフィルタリング
    - P8: リンク抽出・次ページ決定
  - URL フェッチ実装は `plugins.fetcher`（`http` / `chromium`）で選択する。ルーティング、出力、エラーハンドリング、URL正規化は **コア側の固定実装** とする。
- 設定制約
  - 設定はファイル（YAML想定）＋コマンドライン引数で与える。
  - GUI 化時は設定UIから同じ設定構造体を生成できるように、設定はコード上で構造体として完結させる。
- 出力制約
  - 出力先ディレクトリとファイル命名パターンは利用者が設定可能とする（`output.dir` / `output.file_pattern`）。
  - 出力処理自体はコア固定実装とし、プラグインでの差し替えは今回採用しない。

### 受け入れ条件

- `go run ./cmd/<app> -url <URL>` で単一URLが Markdown としてスクレイピングできる。
- 設定ファイルでクロール深度・件数・パスパターン・並行数を指定して複数URLを巡回できる。
- 取得したページ内に PDF へのリンクが含まれる場合、PDF のテキストが抽出される（`parsers` 設定が PDF 有効のとき）。
- プラグインの副作用 import を1行追加するだけで、機能（例: 別のコンテンツ変換器）が有効化される。
- プレゼンテーション層を Fyne に差し替える場合、`usecase` 層以下を変更せずに済む構造になっている。

### 未決事項（後続章で確定）

- HTML→Markdown 変換ライブラリの最終確定
- PDF テキスト抽出ライブラリ・OCRエンジンの最終確定
- 設定ファイルフォーマットの確定（YAML / TOML / JSON）
- ログライブラリの確定（**確定**: 標準 `log/slog` + `internal/logger` で `slog.SetDefault` によりグローバル初期化）
- robots.txt 違反時の挙動（スキップ／エラー／設定で切替）
