## 概要

- 採用した全設定項目（リクエスト制御 / コンテンツ取得 / PDF処理 / クローリング）を Go の構造体として確定する。
- 設定の単一の真実源は `domain/model.Config` 構造体。設定ファイル（YAML）・CLIフラグ・将来のGUIフォームは、すべて同じ構造体に正規化される。
- 各項目のデフォルト値・型・バリデーション・YAML キー・CLI フラグを表で固定し、実装時の判断揺れを排除する。

## 詳細

### 設定構造体の階層

- 設定はカテゴリ別の構造体に分割し、上位の `Config` で集約する。
- 並列ビルド時にプラグイン名で具体実装を選ぶため、`PluginSelection` を別途持つ。

```go
// internal/domain/model/config.go
package model

import "time"

type Config struct {
    Request  RequestConfig   `yaml:"request"`
    Content  ContentConfig   `yaml:"content"`
    PDF      PDFConfig       `yaml:"pdf"`
    Crawl    CrawlConfig     `yaml:"crawl"`
    Plugins  PluginSelection `yaml:"plugins"`
    Targets  []string        `yaml:"targets"` // 起点URL（1件以上）
    Output   OutputConfig    `yaml:"output"`
}

type RequestConfig struct {
    Headers       map[string]string `yaml:"headers"`
    Timeout       time.Duration     `yaml:"timeout"`
    RetryCount    int               `yaml:"retry_count"`
    RetryInterval time.Duration     `yaml:"retry_interval"`
}

type ContentConfig struct {
    Formats         []OutputFormat `yaml:"formats"`
    OnlyMainContent bool           `yaml:"only_main_content"`
    IncludeTags     []string       `yaml:"include_tags"`
    ExcludeTags     []string       `yaml:"exclude_tags"`
    Selector        string         `yaml:"selector"`
    ExtractLinks    bool           `yaml:"extract_links"`
    ExtractMetadata bool           `yaml:"extract_metadata"`
}

type PDFConfig struct {
    Enabled  bool         `yaml:"enabled"`
    Mode     PDFParseMode `yaml:"mode"`
    MaxPages int          `yaml:"max_pages"`
    Output   PDFOutput    `yaml:"output"`
}

type CrawlConfig struct {
    Enabled          bool          `yaml:"enabled"`
    MaxDepth         int           `yaml:"max_depth"`
    MaxPages         int           `yaml:"max_pages"`
    IncludePaths     []string      `yaml:"include_paths"`
    ExcludePaths     []string      `yaml:"exclude_paths"`
    AllowExternal    bool          `yaml:"allow_external_links"`
    AllowSubdomains  bool          `yaml:"allow_subdomains"`
    RequestDelay     time.Duration `yaml:"request_delay"`
    MaxConcurrency   int           `yaml:"max_concurrency"`
    RespectRobotsTxt bool          `yaml:"respect_robots_txt"`
}

type FetcherConfig struct {
    BrowserPath         string        `yaml:"browser_path"`
    UserAgent           string        `yaml:"user_agent"`
    Headless            bool          `yaml:"headless"`
    WaitVisibleSelector string        `yaml:"wait_visible_selector"`
    WaitTimeout         time.Duration `yaml:"wait_timeout"`
}

type PluginSelection struct {
    Fetcher        FetcherKind   `yaml:"fetcher"`         // http / chromium
    FetcherConfig  FetcherConfig `yaml:"fetcher_config"`
    PreProcessors  []string      `yaml:"preprocessors"`   // P2 複数指定可
    Parsers        []string      `yaml:"parsers"`         // P5 複数指定可（HTML/PDF など）
    Transformer    string        `yaml:"transformer"`     // P6 単一
    Filters        []string      `yaml:"filters"`         // P7 複数指定可
    LinkExtractor  string        `yaml:"link_extractor"`  // P8 単一
}

type OutputConfig struct {
    Dir        string `yaml:"dir"`
    FilePattern string `yaml:"file_pattern"`
}
```

### 列挙型の定義

```go
// internal/domain/model/enums.go
package model

type OutputFormat string

const (
    FormatMarkdown OutputFormat = "markdown"
    FormatHTML     OutputFormat = "html"
    FormatRawHTML  OutputFormat = "raw_html"
    FormatJSON     OutputFormat = "json"
    FormatLinks    OutputFormat = "links"
    FormatMetadata OutputFormat = "metadata"
)

type PDFParseMode string

const (
    PDFModeFast PDFParseMode = "fast"
    PDFModeAuto PDFParseMode = "auto"
    PDFModeOCR  PDFParseMode = "ocr"
)

type PDFOutput string

const (
    PDFOutputText     PDFOutput = "text"
    PDFOutputMarkdown PDFOutput = "markdown"
    PDFOutputRaw      PDFOutput = "raw"
)

type FetcherKind string

const (
    FetcherHTTP     FetcherKind = "http"
    FetcherChromium FetcherKind = "chromium"
)
```

### 各設定項目の確定仕様

#### リクエスト制御（RequestConfig）

| 項目 | 型 | YAMLキー | CLIフラグ | デフォルト | 制約 |
| --- | --- | --- | --- | --- | --- |
| カスタムヘッダ | `map[string]string` | `request.headers` | `--header KEY=VAL`（複数可） | 空 | キー・値ともに非空文字列。改行禁止 |
| タイムアウト | `time.Duration` | `request.timeout` | `--timeout` | `60s` | `1s` 以上 `300s` 以下 |
| リトライ回数 | `int` | `request.retry_count` | `--retry` | `2` | `0` 以上 `10` 以下 |
| リトライ間隔 | `time.Duration` | `request.retry_interval` | `--retry-interval` | `1s` | `100ms` 以上 `60s` 以下 |

- URL自体は `targets` または CLI 位置引数で渡す。`RequestConfig` には含めない。

#### コンテンツ取得（ContentConfig）

| 項目 | 型 | YAMLキー | CLIフラグ | デフォルト | 制約 |
| --- | --- | --- | --- | --- | --- |
| 出力フォーマット | `[]OutputFormat` | `content.formats` | `--format`（複数可） | `[markdown]` | 列挙値のみ。重複不可 |
| メインコンテンツのみ | `bool` | `content.only_main_content` | `--only-main` | `true` | - |
| 含めるタグ | `[]string` | `content.include_tags` | `--include-tag` | 空 | HTMLタグ名のみ。`exclude_tags` と同名NG |
| 除外タグ | `[]string` | `content.exclude_tags` | `--exclude-tag` | `[script, style, noscript]` | HTMLタグ名のみ |
| CSSセレクタ | `string` | `content.selector` | `--selector` | `""` | goquery 互換セレクタ。指定時は `only_main_content` より優先 |
| リンク抽出 | `bool` | `content.extract_links` | `--extract-links` | `true`（クロール時は強制 `true`） | - |
| メタデータ抽出 | `bool` | `content.extract_metadata` | `--extract-metadata` | `true` | - |

#### PDF処理（PDFConfig）

| 項目 | 型 | YAMLキー | CLIフラグ | デフォルト | 制約 |
| --- | --- | --- | --- | --- | --- |
| 有効化 | `bool` | `pdf.enabled` | `--pdf` | `true` | `false` の場合、PDF リンクは無視される |
| 解析モード | `PDFParseMode` | `pdf.mode` | `--pdf-mode` | `auto` | `fast` / `auto` / `ocr` のみ |
| 最大ページ数 | `int` | `pdf.max_pages` | `--pdf-max-pages` | `0`（無制限） | `0` 以上 `10000` 以下 |
| 出力形式 | `PDFOutput` | `pdf.output` | `--pdf-output` | `text` | `text` / `markdown` / `raw` のみ |

- リンク中に PDF が含まれる場合、コアが Content-Type または拡張子で判定し PDF パーサーへ振り分ける（詳細は `[[05-パイプライン詳細設計]]`）。

#### クローリング（CrawlConfig）

| 項目 | 型 | YAMLキー | CLIフラグ | デフォルト | 制約 |
| --- | --- | --- | --- | --- | --- |
| 有効化 | `bool` | `crawl.enabled` | `--crawl` | `false` | `false` のとき単一URLモード |
| 最大深度 | `int` | `crawl.max_depth` | `--max-depth` | `2` | `0` 以上 `10` 以下 |
| 最大ページ数 | `int` | `crawl.max_pages` | `--max-pages` | `100` | `1` 以上 `100000` 以下 |
| 含めるパスパターン | `[]string` | `crawl.include_paths` | `--include-path` | 空（=全許可） | 有効なGo正規表現 |
| 除外パスパターン | `[]string` | `crawl.exclude_paths` | `--exclude-path` | 空 | 有効なGo正規表現 |
| 外部リンク追跡 | `bool` | `crawl.allow_external_links` | `--allow-external` | `false` | - |
| サブドメイン追跡 | `bool` | `crawl.allow_subdomains` | `--allow-subdomains` | `false` | - |
| リクエスト間遅延 | `time.Duration` | `crawl.request_delay` | `--delay` | `0s` | `0s` 以上 `60s` 以下。`> 0` の場合 `max_concurrency` は強制1 |
| 並行数 | `int` | `crawl.max_concurrency` | `--concurrency` | `4` | `1` 以上 `64` 以下 |
| robots.txt 準拠 | `bool` | `crawl.respect_robots_txt` | `--respect-robots` | `true` | 判定・取得の詳細は `[[05-パイプライン詳細設計]]` |

#### プラグイン選択（PluginSelection）

| 項目 | 型 | YAMLキー | CLIフラグ | デフォルト | 制約 |
| --- | --- | --- | --- | --- | --- |
| Fetcher | `FetcherKind` | `plugins.fetcher` | `--fetcher` | `http` | `http` / `chromium` のみ（起動時に Fetcher レジストリへ登録済みであること） |
| Fetcher 設定 | `FetcherConfig` | `plugins.fetcher_config` | 下表参照 | headless=true, wait_timeout=5s | chromium 時に使用 |
| P2 PreProcessors | `[]string` | `plugins.preprocessors` | `--preprocessor`（複数可） | 空 | 各名前がレジストリに存在 |
| P5 Parsers | `[]string` | `plugins.parsers` | `--parser`（複数可） | `[html, pdf]` | 各名前がレジストリに存在 |
| P6 Transformer | `string` | `plugins.transformer` | `--transformer` | `markdown` | レジストリに存在 |
| P7 Filters | `[]string` | `plugins.filters` | `--filter`（複数可） | `[maincontent]` | 各名前がレジストリに存在 |
| P8 LinkExtractor | `string` | `plugins.link_extractor` | `--link-extractor` | `default` | レジストリに存在 |

#### Fetcher 専用設定（FetcherConfig、`plugins.fetcher: chromium` 時）

| 項目 | 型 | YAMLキー | CLIフラグ | デフォルト | 制約 |
| --- | --- | --- | --- | --- | --- |
| ブラウザパス | `string` | `fetcher_config.browser_path` | `--fetcher-browser-path` | 空（自動検出） | 実行可能ファイル |
| User-Agent | `string` | `fetcher_config.user_agent` | `--fetcher-user-agent` | 空 | 改行禁止。空時は request.headers または既定 UA |
| ヘッドレス | `bool` | `fetcher_config.headless` | `--fetcher-headless` | `true` | - |
| 待機セレクタ | `string` | `fetcher_config.wait_visible_selector` | - | 空 | CSS セレクタ。空なら待機しない |
| 待機タイムアウト | `time.Duration` | `fetcher_config.wait_timeout` | - | `5s` | `0s` 以上 `120s` 以下 |

#### 出力設定（OutputConfig）

| 項目 | 型 | YAMLキー | CLIフラグ | デフォルト | 制約 |
| --- | --- | --- | --- | --- | --- |
| 出力ディレクトリ | `string` | `output.dir` | `--output-dir` | `./out` | 書き込み可能なパス |
| ファイル命名パターン | `string` | `output.file_pattern` | `--output-pattern` | `{seq}-{host}-{path}.{ext}` | `{seq}` `{host}` `{path}` `{ext}` のみ有効 |

- `output` 系項目は当面の動作確認のための最小仕様。本格的な出力先切替はプラグイン化していないため、コア側固定実装でディスク書込のみサポート。

### バリデーション仕様

- バリデーションは `model.Config` に `Validate() error` メソッドを生やして集中管理する。
- バリデーション項目:
    1. `Targets` が1件以上、各要素が `http://` または `https://` で始まること。
    2. `RequestConfig.Timeout` `RetryCount` `RetryInterval` が表の範囲内。
    3. `ContentConfig.Formats` が列挙値、重複なし。
    4. `ContentConfig.IncludeTags` と `ExcludeTags` に同名がない。
    5. `ContentConfig.Selector` が指定された場合、goquery でパース可能であること（試験パース）。
    6. `PDFConfig.Mode` `Output` が列挙値。
    7. `CrawlConfig.MaxDepth` `MaxPages` `MaxConcurrency` `RequestDelay` が範囲内。
    8. `CrawlConfig.IncludePaths` `ExcludePaths` 各要素が `regexp.Compile` 可能。
    9. `CrawlConfig.RequestDelay > 0` の場合は `MaxConcurrency = 1` に強制（警告ログ）。
    10. `PluginSelection.Fetcher` が `http` / `chromium` のいずれかであること（列挙値チェック。実際の生成は `Kernel.Init` でレジストリ参照）。
    11. `PluginSelection.FetcherConfig.WaitTimeout` が `0s` 以上 `120s` 以下であること。
    12. `PluginSelection` の P2〜P8 各名前が、プラグインレジストリに登録済みであること（カーネル `Init` 時にチェック）。Fetcher は項目 10 と生成時のレジストリ参照で検証する。
    13. `OutputConfig.FilePattern` に未知のプレースホルダがないこと。
- エラーは `errors.Join` で集約し、すべての違反を一度に通知する。

### デフォルト値の適用順序

1. ゼロ値で `Config` を構築。
2. 内部の `defaultConfig()` を適用（上表のデフォルト値）。
3. 設定ファイル（YAML）でマージ。
4. CLI フラグでマージ（最終優先度）。
5. `Validate()` 実行。

### 設定ファイル例

```yaml
# configs/config.example.yaml
targets:
  - https://example.com/docs

request:
  headers:
    User-Agent: "scraperbot/0.1"
  timeout: 60s
  retry_count: 2
  retry_interval: 1s

content:
  formats: [markdown, links]
  only_main_content: true
  include_tags: []
  exclude_tags: [script, style, noscript]
  selector: ""
  extract_links: true
  extract_metadata: true

pdf:
  enabled: true
  mode: auto
  max_pages: 0
  output: text

crawl:
  enabled: true
  max_depth: 2
  max_pages: 100
  include_paths:
    - "^/docs/.*"
  exclude_paths:
    - ".*\\.zip$"
  allow_external_links: false
  allow_subdomains: false
  request_delay: 0s
  max_concurrency: 4
  respect_robots_txt: true

plugins:
  fetcher: http
  fetcher_config:
    browser_path: ""
    user_agent: ""
    headless: true
    wait_visible_selector: ""
    wait_timeout: 5s
  preprocessors: []
  parsers: [html, pdf]
  transformer: markdown
  filters: [maincontent]
  link_extractor: default

output:
  dir: "./out"
  file_pattern: "{seq}-{host}-{path}.{ext}"
```

### CLI フラグ設計の補足

- 単純な値は `--key value` 形式。
- 繰り返し可フラグは同名フラグを複数回指定可能（例: `--format markdown --format links`）。
- ヘッダは `KEY=VAL` で1回ごとに1ヘッダ（例: `--header User-Agent=scraperbot/0.1`）。
- フラグの省略時は YAML 設定 → 既定値 の順でフォールバック。

### 設定ロード処理の責務分割

- `internal/infrastructure/configloader/yaml.go`: YAMLファイル→構造体への変換。
- `internal/presentation/cli/flags.go`: CLIフラグ→構造体への変換。
- `internal/usecase/config_merger.go`: 既定値・YAML・CLI のマージと最終 `Validate()` 呼び出し。
- GUI 化時は `internal/presentation/gui/form.go` が同じ構造体を生成して `config_merger` の最終マージに渡す。

### 受け入れ条件

- 上記すべての項目に対して、構造体定義・YAMLキー・CLIフラグ・デフォルト値・制約が表で示されている。
- `Validate()` がすべての検証ルールを一括チェックし、エラーは集約して返す。
- YAML設定とCLIフラグの両方から最終 `Config` を構築でき、最終構造は GUI からも同様に作れる。
- `PluginSelection` で指定したプラグイン名は、カーネル `Init` 時にレジストリ存在チェックされる。
